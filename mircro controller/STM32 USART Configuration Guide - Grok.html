To change from USART1 to another USART (e.g., USART2, USART3, etc.) on an STM32 microcontroller, you need to modify the code to account for the new USART's base address, clock enable bit, pin configuration, and alternate function settings. Below is a step-by-step guide to adapt your code:

1. Identify the New USART and Its Base Address

Each USART peripheral has a unique base address in the STM32 memory map. Check the reference manual for your specific STM32 microcontroller to find the base address of the desired USART. For example, common base addresses (for STM32F4 series) are:

  • USART1: 0x40011000
  • USART2: 0x40004400
  • USART3: 0x40004800
  • UART4: 0x40004C00
  • UART5: 0x40005000
  • USART6: 0x40011400

Update the base address definition in your code. For example, to switch to USART2:

c
#define UART2_BASE (0x40004400)

Then, update all register definitions to use the new base address:

c
#define UART2_SR (*(volatile unsigned int *)(UART2_BASE + 0x00)) #define UART2_BRR (*(volatile unsigned int *)(UART2_BASE + 0x08)) #define UART2_CR1 (*(volatile unsigned int *)(UART2_BASE + 0x0C)) #define UART2_DR (*(volatile unsigned int *)(UART2_BASE + 0x04))

Replace UART1 with UART2 (or the chosen USART) in all register names and function references.

2. Update Clock Enable for the New USART

The clock for each USART is enabled in different registers depending on the peripheral:

  • USART1 and USART6 are typically on the APB2 bus.
  • USART2, USART3, UART4, UART5 are typically on the APB1 bus.

Check the RCC (Reset and Clock Control) section of the reference manual to confirm the correct register and bit for enabling the clock. For example:

  • For USART2 (APB1 bus), enable the clock in RCC_APB1ENR:
    c
    #define RCC_APB1ENR (*(volatile unsigned int *)(RCC_BASE + 0x40)) RCC_APB1ENR |= (1 << 17); // Enable USART2 clock
  • For USART3 (APB1 bus):
    c
    RCC_APB1ENR |= (1 << 18); // Enable USART3 clock

In your code, replace the USART1 clock enable line:

with the appropriate line for the new USART, e.g., for USART2:

c
RCC_APB1ENR |= (1 << 17); // USART2

3. Update GPIO Pin Configuration

Each USART uses specific pins for TX and RX, and these pins must be configured for the alternate function (AF) corresponding to the new USART. Check the alternate function mapping in the microcontrollers datasheet to find:

  • The pins associated with the new USARTs TX (and RX, if needed).
  • The alternate function code (e.g., AF7 or AF8) for the USART.

For example:

  • USART1: TX on PA9 (AF7).
  • USART2: TX on PA2 (AF7) or PD5 (AF7).
  • USART3: TX on PB10 (AF7) or PD8 (AF7).

If switching to USART2 with TX on PA2:

And update the register definitions and bit positions accordingly.

4. Update Baud Rate and USART Configuration

The baud rate calculation and USART control register settings (UARTx_CR1, UARTx_BRR, etc.) remain the same unless the new USART is on a different bus with a different clock frequency:

  • APB1 (USART2, USART3, etc.) may have a different clock frequency than APB2 (USART1, USART6).
  • Check the system clock configuration to determine the APB1/APB2 clock frequency (e.g., SYSCLK may be divided by a prescaler).

Rename the functions to reflect the new USART, e.g., uart1_tx_init to uart2_tx_init and uart1_send to uart2_send. Update all references in main and function definitions.
